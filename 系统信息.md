## 一、系统概述

**系统名称**：港好住信息系统平台
**所属单位**：郑州航空港区
**系统定位**：面向港区保租房、人才公寓和市场租赁的全流程数字化管理平台
**业务范围**：覆盖资格审核、选房、签约、入住、租金缴费、续租、退租、开票等全生命周期业务

**服务对象**：
- **租户（用户端）**：通过 H5/小程序 办理各类租住业务
- **管理员（后台端）**：通过 PC 管理端进行房源管理、合同审批、账单催缴等
- **企业客户**：通过企业入口进行员工集中配租管理

## 三、系统架构

### 3.1 整体架构

采用**前后端分离**架构：

```
┌─────────────────────────────────────────────────────┐
│                    客户端层                          │
│   管理端 PC (Vue 2 + Element UI)                    │
│   用户端 H5/小程序 (uni-app + Vue 2)                │
└─────────────────────┬───────────────────────────────┘
                       │ HTTP/HTTPS  JWT Token
┌─────────────────────▼───────────────────────────────┐
│                   后端服务层                         │
│   Spring Boot 3.5.4 + Spring Security               │
│   RESTful API  /  Swagger UI                        │
│   端口：8090                                        │
└────────┬──────────────────────┬────────────────────┘
         │                      │
┌────────▼─────────┐  ┌────────▼──────────┐
│  达梦数据库(DM8) │  │   Redis 缓存       │
│  newgangzhu      │  │   localhost:6379   │
└──────────────────┘  └───────────────────┘
```

## 十、开发规范摘要

### 10.1 后端开发规范

1. **ORM**：使用 MyBatis-Plus，继承 `BaseMapper<T>`，简单 CRUD 无需编写 XML
2. **Controller**：使用 `@RestController` + `@RequestMapping`，统一返回 `AjaxResult`
3. **Service**：接口与实现分离，实现类继承 `ServiceImpl<M, T>`
4. **全局异常**：通过 `GlobalExceptionHandler` 统一处理
5. **逻辑删除**：所有表使用 del_flag 字段（0=正常，2=删除），MyBatis-Plus 自动处理
6. **主键策略**：BIGINT AUTO_INCREMENT
7. **代码位置**：业务代码尽量写在 ruoyi-admin 模块

### 10.2 达梦数据库适配注意事项

从 MySQL 迁移到达梦数据库时，需注意以下兼容性问题：

| MySQL 语法 | 达梦 DM8 适配方案 |
|-----------|----------------|
| `ENGINE=InnoDB` | 删除（达梦不需要） |
| `AUTO_INCREMENT` | 使用 `IDENTITY(1,1)` 或序列 |
| `TINYINT(1)` | 改为 `SMALLINT` |
| `TEXT/LONGTEXT` | 改为 `CLOB` |
| `BLOB` | 改为 `BLOB`（兼容） |
| `GROUP_CONCAT` | 使用 `WM_CONCAT` 或 `LISTAGG` |
| `IFNULL` | 改为 `COALESCE` 或 `NVL` |
| `DATE_FORMAT` | 改为 `TO_CHAR(date, 'YYYY-MM-DD')` |
| `LIMIT` 分页 | MyBatis-Plus 分页插件自动处理 |
| 字符集 `utf8mb4` | 达梦默认支持 Unicode |

### 10.3 前端开发规范（管理端）

1. API 接口统一在 `src/api/` 目录定义
2. 使用 `@/` 别名引用 `src` 目录
3. 遵循 Element UI 组件规范
4. 图片显示：数据库存相对路径，前端拼接 `/dev-api` 前缀显示

### 10.4 前端开发规范（用户端 UniApp）

1. API 地址集中配置在 `config/index.js`，禁止硬编码
2. 图片路径使用 `getImageUrl()` 方法处理
3. 需要登录的页面必须在 `onLoad` 中使用 `authCheck.checkLogin` 检查
4. 跳转登录页必须使用 `uni.redirectTo`（避免返回循环）
