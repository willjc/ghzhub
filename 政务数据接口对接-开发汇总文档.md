> **状态**：阶段二（代理服务开发）已完成，阶段一/三/四/五待开发

## 一、项目概述

港好住人才公寓申请系统需要调用 5 个政务数据接口（婚姻、社保、公租房、不动产、入住记录），实现自动资格校验。由于政务数据交换平台（`59.207.50.51:4989`）仅在**电子政务外网**可访问，业务服务器在互联网上无法直连，因此引入中间代理层。

### 1.1 架构

```
┌──────────────────┐     HTTP(互联网)    ┌────────────────────┐     SDK加密(政务外网)    ┌──────────────────┐
│  港好住业务系统    │ ──────────────────▶ │  接口代理服务        │ ──────────────────────▶ │  数据交换平台      │
│  (ruoyi-admin)   │                    │  ghz-gov-proxy     │                        │  59.207.50.51:4989│
│  互联网服务器      │                    │  政务外网服务器       │                        │                  │
│  端口 8090        │                    │  端口 9001          │                        │                  │
└──────────────────┘                    └────────────────────┘                        └──────────────────┘
```

**两层直连**，无中间 Nginx 层。网络由运维打通（业务服务器 → 政务外网服务器 IP）。

### 1.2 涉及的两个子项目

| 项目 | 路径 | 用途 | 部署位置 |
|------|------|------|----------|
| `ghz-gov-proxy` | `ghz-gov-proxy/` | 独立的 Spring Boot 代理服务 | 政务外网电脑 |
| `ruoyi-system/gov/` | `ruoyi-system/src/main/java/com/ruoyi/system/gov/` | 业务系统中的 HTTP 客户端 | 业务服务器（随主项目部署） |

### 1.3 5 个政务接口一览

| 接口 | 请求方式 | 核心参数 | 用途 |
|------|---------|---------|------|
| 婚姻查询 | SDK POST | sfzh（身份证）、xm（姓名） | 核实婚姻状态、获取配偶信息 |
| 社保查询 | SDK POST | AAB004（单位名称）、AAB039（信用代码） | 核实单位是否正常缴纳社保 |
| 公租房查询 | SDK POST | card_num（身份证）、name（姓名） | 核实是否已享受公租房保障 |
| 不动产查询 | SDK POST | personCertNo（身份证）、personName（姓名） | 核实名下是否有房产 |
| 入住记录 | 本地数据库 | - | 查 `hz_checkin_record` 状态 2/4 |

### 2.3 Token 管理机制

- **鉴权模式**：OAuth2 `grant_type=client_credentials`
- **有效期**：1800 秒（30 分钟）
- **刷新策略**：定时任务每 25 分钟刷新，提前 5 分钟（300 秒）判定过期
- **并发安全**：`volatile` + `synchronized(refreshLock)` 双重检查锁
- **SDK 密钥**：
  - AppID: `b2619ca04a2446dea85f741328ddc238`
  - AppSecret: `763F673675DE32D1DFA9D2F2F11A2916`
- **Token 响应格式**：
  ```json
  {"controls":[],"custom":{"access_token":"xxx","expires_in":"1800"},"status":{"code":"1"}}
  ```

**所有 4 个业务接口共用同一个 access_token。**

### 2.4 代理服务配置文件

```yaml
# application.yml
server:
  port: 9001

gov:
  proxy:
    api-key: ghz-gov-proxy-key-2026

logging:
  level:
    com.ghz.gov: DEBUG
```

### 2.5 打包与部署

#### 打包

```bash
cd ghz-gov-proxy
mvn clean package -DskipTests
# 产出：target/ghz-gov-proxy-1.0.0.jar
```

#### 部署到政务外网电脑

1. **环境要求**：JRE 1.8+（不需要 JDK、不需要 Maven）
2. **启动命令**：
   ```bash
   java -jar ghz-gov-proxy-1.0.0.jar
   ```
3. **验证启动成功**（在政务电脑本地执行）：
   ```bash
   curl -H "X-Api-Key: ghz-gov-proxy-key-2026" http://localhost:9001/api/v1/gov/token/status
   ```
   返回 `{"code":200,"data":{"hasToken":true,"isValid":true,...}}` 即成功。
4. 如果 `hasToken` 为 false，说明到 `59.207.50.51:4989` 网络不通，需排查网络。

## 三、代理服务对外接口（5个）

> 所有接口统一响应格式：`{"code":200,"message":"success","data":{...}}`  
> 认证方式：请求头 `X-Api-Key: ghz-gov-proxy-key-2026`

### 3.2 婚姻查询

| 项 | 值 |
|----|-----|
| 方法 | `POST` |
| 路径 | `/api/v1/gov/marriage/query` |
| Content-Type | `application/json` |

**请求体**：
```json
{"idCard": "410102199001011234", "name": "张三"}
```

**响应 data 字段**：
```json
{
  "hasRecord": true,
  "raw": {
    "女方姓名": "李四",
    "女方身份证件号码": "410102199002022345",
    "男方姓名": "张三",
    "男方身份证件号码": "410102199001011234",
    "登记日期": "2020-05-20"
  }
}
```

**注意事项**：
- 未婚时 `hasRecord: false`，`raw` 为空对象 `{}`
- 返回的 JSON key 是中文，不要用英文字段名去取
- 需要根据申请人性别判断取 `女方` 还是 `男方` 的字段作为配偶信息

**实际调政务平台的请求体**（代理服务内部构造）：
```json
{"sfzh": "身份证号", "xm": "姓名"}
```

### 3.3 社保查询

| 项 | 值 |
|----|-----|
| 方法 | `POST` |
| 路径 | `/api/v1/gov/social/query` |

**请求体**：
```json
{"companyName": "郑州航空港区某某公司", "creditCode": "91410100MA3XABCD12"}
```

**响应 data 字段**：
```json
{
  "hasRecord": true,
  "records": [
    {
      "AAB004": "单位名称",
      "AAB039": "统一信用代码",
      "AAB083": "缴费人数",
      "AAE041": "缴费年月",
      "AAE140": "险种类型"
    }
  ],
  "raw": { ... }
}
```

**注意事项**：
- 校验逻辑不是"这个人有没有社保"，而是"这个单位是否在正常缴纳社保"（单位存在性校验）
- `creditCode`（统一社会信用代码）+ `companyName`（单位名称）都必填，用于 `AAB039` 参数
- `STARTDATE`/`ENDDATE` 目前传空字符串

**实际调政务平台的请求体**（代理服务内部构造）：
```json
{"AAB004": "单位名称", "AAB039": "信用代码", "STARTDATE": "", "ENDDATE": ""}
```

```yaml

  proxy:
    base-url: http://127.0.0.1:9001   # ← 部署时改为政务外网服务器IP:9001
    api-key: ghz-gov-proxy-key-2026
```

### 4.3 GovDataClient 方法一览

| Java 方法 | 对应接口 | 参数 |
|-----------|---------|------|
| `queryMarriage(idCard, name)` | 婚姻查询 | String idCard, String name |
| `querySocialInsurance(companyName, creditCode)` | 社保查询 | String companyName, String creditCode |
| `queryPublicHousing(idCard, name)` | 公租房查询 | String idCard, String name |
| `queryRealEstate(idCard, name)` | 不动产查询 | String idCard, String name |
| `tokenStatus()` | Token状态 | 无 |

### 4.4 返回值格式

所有查询方法返回 `Map<String, Object>`，包含：

| key | 类型 | 说明 |
|-----|------|------|
| `success` | boolean | 调用是否成功（false 表示网络异常或代理返回错误） |
| `error` | String | 失败时的错误信息（success=false 时存在） |
| `hasRecord` | boolean | 政务平台是否有查到记录（success=true 时有） |
| `raw` | JSONObject | 原始返回数据（success=true 时有） |
| `records` | List | 社保接口返回的缴费记录列表 |

### 4.5 业务代码调用示例

```java
@Service
public class SomeService {

    @Autowired
    private GovDataClient govDataClient;

    public void checkSomething(String idCard, String name, String companyName, String creditCode) {
        // 1. 查婚姻
        Map<String, Object> marriage = govDataClient.queryMarriage(idCard, name);
        if (!(boolean) marriage.get("success")) {
            log.error("婚姻查询失败: {}", marriage.get("error"));
            return;
        }
        boolean married = (boolean) marriage.get("hasRecord");

        // 2. 查社保
        Map<String, Object> social = govDataClient.querySocialInsurance(companyName, creditCode);
        boolean hasSocial = (boolean) social.get("success") && (boolean) social.get("hasRecord");

        // 3. 查公租房
        Map<String, Object> housing = govDataClient.queryPublicHousing(idCard, name);
        boolean hasHousing = (boolean) housing.get("success") && (boolean) housing.get("hasRecord");

        // 4. 查不动产
        Map<String, Object> estate = govDataClient.queryRealEstate(idCard, name);
        boolean hasEstate = (boolean) estate.get("success") && (boolean) estate.get("hasRecord");

        // 5. 综合判断...
    }
}
```
