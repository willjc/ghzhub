> 编制日期：2026年4月  
> 版本：V1.0

---

## 一、项目背景与需求分析

### 1.1 业务背景

港好住人才公寓申请流程中，需要核实申请人是否符合以下条件：

| 序号 | 审核条件 | 依赖接口 | 判断逻辑 |
|------|---------|---------|---------|
| 1 | 夫妻双方只能申请一套人才公寓 | 省民政_婚姻实时信息V2 | 查询婚姻状态，关联配偶身份，校验配偶是否已有申请 |
| 2 | 郑州市航空港区内不能有住房 | 不动产登记信息V1 | 查询不动产记录，判断港区内是否有房产 |
| 3 | 名下没有公租房 | 公租房信息接口 | 查询公租房配租记录，判断是否有在租公租房 |
| 4 | 社保信息中投保单位跟填写的单位名称一致 | 省人社_参保单位社保缴费信息V7 | 查询社保缴费单位，与用户填写的单位名称比对 |

### 1.2 数据来源

以上数据均来自 **航空港区数据交换平台**（电子政务外网），需通过省级数据共享交换体系获取。

### 1.3 核心难点

- 数据接口位于 **电子政务外网**，无法从互联网直接访问
- 需经过 **两层网络跳转** 才能从我们的服务器调通政务接口
- Token 有效期仅 30 分钟，需设计自动刷新机制
- 涉及个人隐私数据（身份证、婚姻、房产），需保证传输安全

## 三、接口需求明细

### 3.1 数据交换平台接口清单

所有接口基础地址：`http://59.207.50.51:4989`，协议：POST/JSON，网络：电子政务外网

#### 3.1.1 获取访问令牌（Token）

| 项目 | 说明 |
|------|------|
| 接口地址 | `http://59.207.50.51:4989` |
| 提交方式 | POST |
| Content-Type | `application/x-www-form-urlencoded` |
| 功能 | 获取 access_token，后续接口调用必备 |

**输入参数：**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| client_id | String | 是 | 应用ID（申请后分配） |
| client_secret | String | 是 | 应用密钥（申请后分配） |
| grant_type | String | 是 | 固定值 `client_credentials` |

**输出参数：**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| access_token | String | 访问令牌，有效期30分钟 |
| refresh_token | String | 刷新令牌 |
| expires_in | int | 过期时间（秒），默认1800 |

**返回示例：**
```json
{
  "controls": [],
  "custom": {
    "access_token": "e4e25214e9d7c9725d270eb17a932ad8",
    "refresh_token": "16cccc026a2552f7eed8693797204a2b",
    "jsessionid": "",
    "extendinfo": "",
    "expires_in": "1800"
  },
  "status": {"code": "1"}
}
```

### 4.2 程序组件规划

整个对接方案涉及 **三个程序组件**：

| 组件 | 部署位置 | 功能 | 技术选型 |
|------|---------|------|---------|
| **政务数据API代理服务** | 172.19.25.77 | 封装政务接口调用，对外提供标准RESTful API | Spring Boot 2.x |
| **Nginx反向代理** | 172.27.139.244 | 纯网络层转发，将请求透传到政务外网 | Nginx |
| **政务数据Client模块** | 港好住业务服务器 | 封装对代理服务的调用，供业务层使用 | Java (集成到ruoyi-system) |

> **为什么需要API代理服务而不是直接透传？**
> - 政务接口的调用方式（Token获取、参数格式、鉴权机制）与标准RESTful差异较大
> - Token管理（获取、缓存、刷新）应在离数据源最近的地方处理，减少跨网络Token传递
> - 代理服务可以对敏感数据做脱敏、审计日志记录
> - 代理服务可以做限流、熔断，保护政务接口不被过载

#### 5.1.3 Token管理机制

```
                    ┌─────────────────────────────┐
                    │        TokenService          │
                    │                              │
                    │  accessToken: String         │
                    │  refreshToken: String        │
                    │  expiresAt: Long             │
                    │  lock: ReentrantLock         │
                    │                              │
                    │  getToken(): String           │
                    │  refreshToken(): void         │
                    │  scheduledRefresh(): void     │
                    └─────────────────────────────┘
                               │
          ┌────────────────────┼────────────────────┐
          │                    │                    │
          ▼                    ▼                    ▼
    首次调用时获取       定时刷新(25分钟)      过期前5分钟
    (懒加载)            (ScheduledTask)       自动续期
```

**Token管理策略：**
- **懒加载**：服务启动后不立即获取Token，首次调用时获取
- **定时刷新**：每25分钟自动刷新一次（Token有效期30分钟）
- **过期检测**：每次调用前检查Token是否过期，过期则重新获取
- **并发安全**：使用 ReentrantLock 保证同一时刻只有一个线程获取/刷新Token
- **异常重试**：Token获取失败时重试3次，间隔5秒

```java
/**
 * 资格审核综合判定逻辑
 */
public QualificationResultDTO checkQualification(Long applyId) {
    QualificationResultDTO result = new QualificationResultDTO();
    
    // 1. 获取申请人信息
    HzApply apply = applyMapper.selectById(applyId);
    String idCard = apply.getIdCard();
    String name = apply.getName();
    String companyName = apply.getCompanyName();
    String creditCode = apply.getCreditCode();
    
    // 2. 婚姻校验 — 夫妻双方只能申请一套
    MarriageInfoDTO marriage = govDataClient.queryMarriage(idCard, name);
    if (marriage.isMarried() && marriage.getSpouseIdCard() != null) {
        // 查询配偶是否已有人才公寓申请
        boolean spouseApplied = applyMapper.existsSpouseApply(marriage.getSpouseIdCard());
        result.setMarriageCheck(spouseApplied ? "不通过" : "通过");
        result.setMarriageRemark(spouseApplied ? "配偶已有人才公寓申请" : "配偶无申请记录");
    } else {
        result.setMarriageCheck("通过");
        result.setMarriageRemark("未婚或无配偶信息");
    }
    
    // 3. 不动产校验 — 航空港区内不能有住房
    RealEstateDTO realEstate = govDataClient.queryRealEstate(idCard, name);
    boolean hasGangQuEstate = realEstate.isHasRealEstate();
    result.setRealEstateCheck(hasGangQuEstate ? "不通过" : "通过");
    result.setRealEstateRemark(hasGangQuEstate ? "在航空港区有房产登记" : "无房产登记");
    
    // 4. 公租房校验 — 名下不能有公租房
    PublicHousingDTO publicHousing = govDataClient.queryPublicHousing(idCard, name);
    boolean hasPublicHousing = publicHousing.isHasPublicHousing();
    result.setPublicHousingCheck(hasPublicHousing ? "不通过" : "通过");
    result.setPublicHousingRemark(hasPublicHousing ? "名下有公租房" : "无公租房记录");
    
    // 5. 社保校验 — 投保单位与填写单位一致
    SocialInsuranceDTO insurance = govDataClient.querySocialInsurance(companyName, creditCode);
    boolean companyMatch = insurance != null && companyName.equals(insurance.getCompanyName());
    result.setSocialInsuranceCheck(companyMatch ? "通过" : "不通过");
    result.setSocialInsuranceRemark(companyMatch ? "社保投保单位与填写单位一致" : "社保投保单位与填写单位不一致");
    
    // 6. 综合判定
    boolean qualified = "通过".equals(result.getMarriageCheck())
        && "通过".equals(result.getRealEstateCheck())
        && "通过".equals(result.getPublicHousingCheck())
        && "通过".equals(result.getSocialInsuranceCheck());
    result.setQualified(qualified);
    
    return result;
}
```

### 6.3 API Key 设计

```java
// API Key格式: appKey:timestamp:sign
// sign = MD5(appKey + timestamp + secretKey)
// 示例:
// X-Api-Key: ghz_app:1713672000000:a1b2c3d4e5f6g7h8

@Component
public class ApiKeyFilter extends OncePerRequestFilter {
    
    @Value("${security.api-keys}")
    private List<String> validApiKeys;
    
    @Override
    protected void doFilterInternal(HttpServletRequest request, 
            HttpServletResponse response, FilterChain filterChain) {
        String apiKey = request.getHeader("X-Api-Key");
        if (apiKey == null || !validApiKeys.contains(apiKey)) {
            response.setStatus(401);
            return;
        }
        filterChain.doFilter(request, response);
    }
}
```
