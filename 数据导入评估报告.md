### 2.2 ⚠️ 需要关注的问题

#### 问题 1：租户数据量异常庞大（128,032 条）

**现象：**
- 租户数：128,032
- 合同数：3,655
- 比例：35:1（平均每个合同对应 35 个租户记录）

**分析：**
这明显不合理。可能的原因：
1. 老系统存在大量历史测试数据
2. 租户信息未做去重处理
3. 同一租户多次录入（换房、续租时重复创建）

**影响：**
- 租户管理页面加载缓慢（已通过分页修复）
- 数据库查询性能下降
- 存储空间浪费

**建议：**
```sql
-- 检查重复租户（相同身份证号）
SELECT id_card, COUNT(*) as count 
FROM hz_tenant 
WHERE del_flag = '0' 
GROUP BY id_card 
HAVING count > 1 
ORDER BY count DESC 
LIMIT 20;

-- 检查是否有大量无合同关联的租户
SELECT COUNT(*) 
FROM hz_tenant t
LEFT JOIN hz_contract c ON t.tenant_id = c.tenant_id AND c.del_flag = '0'
WHERE t.del_flag = '0' AND c.contract_id IS NULL;
```

**处理方案：**
1. 保留有合同关联的租户
2. 对于重复身份证号的租户，合并为一条记录
3. 删除无任何关联的孤立租户数据

#### 问题 2：异常租金价格

**现象：**
- 最低租金：0.10 元
- 最高租金：2,430.46 元
- 平均租金：657.93 元
- 异常低价合同：2 条（< 100 元）

**样本数据：**
```

```

**分析：**
- 0.10 元的租金明显是测试数据
- 需要确认这些合同是否应该保留

**建议：**
```sql

SELECT contract_id, contract_no, tenant_id, house_id, 
       rent_price, start_date, end_date, contract_status

```

**处理方案：**
- 如果是测试数据，建议删除
- 如果是特殊优惠合同，需要在备注中说明

#### 问题 4：合同状态分布不均

**现象：**
- 状态 3（履行中）：912 条（24.95%）
- 状态 5（已解约）：2,743 条（75.05%）

**分析：**
- 75% 的合同已解约，说明是历史数据为主
- 仅 25% 的合同在履行中

**影响：**
- 如果这些历史数据不需要，会占用大量存储
- 建议确认是否需要保留所有已解约合同

---

#### 问题 5：账单未缴费比例高

**现象：**
- 未缴费（status=0）：80,187 条（67.47%）
- 已缴费（status=1）：38,656 条（32.53%）
- 部分缴费（status=2）：5 条（0.00%）

**分析：**
- 67% 的账单未缴费，可能包含：
  - 已解约合同的历史欠费
  - 未来账单（预生成）
  - 实际欠费

**建议：**
```sql
-- 检查未缴费账单的合同状态分布
SELECT c.contract_status, COUNT(*) as unpaid_bills
FROM hz_bill b
JOIN hz_contract c ON b.contract_id = c.contract_id
WHERE b.del_flag = '0' AND b.bill_status = '0' AND c.del_flag = '0'
GROUP BY c.contract_status;

-- 检查未缴费账单的到期时间分布
SELECT 
    CASE 
        WHEN due_date < CURDATE() THEN '已逾期'
        WHEN due_date = CURDATE() THEN '今日到期'
        WHEN due_date > CURDATE() THEN '未到期'
    END as status,
    COUNT(*) as count
FROM hz_bill
WHERE del_flag = '0' AND bill_status = '0'
GROUP BY status;
```

## 六、SQL 检查脚本

```sql
-- 1. 检查租户重复情况
SELECT id_card, COUNT(*) as count, GROUP_CONCAT(tenant_id) as tenant_ids
FROM hz_tenant 
WHERE del_flag = '0' 
GROUP BY id_card 
HAVING count > 1 
ORDER BY count DESC 
LIMIT 50;

-- 2. 检查无合同关联的租户
SELECT COUNT(*) as orphan_tenants
FROM hz_tenant t
LEFT JOIN hz_contract c ON t.tenant_id = c.tenant_id AND c.del_flag = '0'
WHERE t.del_flag = '0' AND c.contract_id IS NULL;

-- 3. 检查异常价格合同
SELECT contract_id, contract_no, tenant_id, house_id, 
       rent_price, deposit, start_date, end_date
FROM hz_contract 
WHERE del_flag = '0' AND (rent_price < 100 OR rent_price > 10000)
ORDER BY rent_price;

-- 4. 检查零金额账单
SELECT bill_id, contract_id, bill_type, bill_amount, 
       bill_status, due_date, create_time
FROM hz_bill 
WHERE del_flag = '0' AND bill_amount <= 0;

-- 5. 检查逾期未缴费账单
SELECT COUNT(*) as overdue_bills, SUM(bill_amount) as overdue_amount
FROM hz_bill
WHERE del_flag = '0' 
  AND bill_status = '0' 
  AND due_date < CURDATE();

-- 6. 检查合同与账单的一致性
SELECT c.contract_id, c.contract_no, 
       COUNT(b.bill_id) as bill_count,
       SUM(CASE WHEN b.bill_status = '1' THEN b.bill_amount ELSE 0 END) as paid_amount
FROM hz_contract c
LEFT JOIN hz_bill b ON c.contract_id = b.contract_id AND b.del_flag = '0'
WHERE c.del_flag = '0' AND c.contract_status = '3'
GROUP BY c.contract_id
HAVING bill_count = 0;
```
