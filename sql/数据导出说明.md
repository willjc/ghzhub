## 一、使用方法

### 方法1: 使用 MySQL 客户端导出

```bash
# 登录 MySQL
mysql -u root -p newgangzhu

# 执行导出脚本
source /path/to/export_old_data.sql

# 或者直接执行
mysql -u root -p newgangzhu < export_old_data.sql > output.txt
```

### 方法2: 使用 Navicat/DBeaver 等工具

1. 打开工具，连接到 `newgangzhu` 数据库
2. 新建查询窗口
3. 复制 `export_old_data.sql` 中的内容
4. 执行查询，导出结果

### 方法3: 使用 mysqldump 导出整个表

```bash
# 导出单个表(包含结构+数据)
mysqldump -u root -p newgangzhu hz_project > hz_project.sql

# 只导出数据(不包含表结构)
mysqldump -u root -p -t newgangzhu hz_project > hz_project_data.sql

# 导出多个表
mysqldump -u root -p newgangzhu hz_project hz_building hz_house > base_data.sql

# 导出所有业务表(hz_开头)
mysqldump -u root -p newgangzhu hz_* > business_data.sql
```

### 方法4: 导出为 Excel/CSV 格式

```bash
# 使用 MySQL 导出为 CSV
mysql -u root -p -e "SELECT * FROM hz_project" newgangzhu > hz_project.csv

# 或在查询中添加导出语句
SELECT * FROM hz_project INTO OUTFILE '/tmp/hz_project.csv'
FIELDS TERMINATED BY ',' ENCLOSED BY '"'
LINES TERMINATED BY '\n';
```

## 五、注意事项

### 5.1 图片文件处理

数据库中存储的是**相对路径**，如 `/profile/upload/2025/01/19/xxx.jpg`

需要同步导出服务器上的实际文件：
```
D:/ruoyi/uploadPath/upload/
```

### 5.2 关联关系

导出时注意保持表之间的关联关系：
- `hz_building.project_id` → `hz_project.project_id`
- `hz_house.building_id` → `hz_building.building_id`
- `hz_contract.tenant_id` → `hz_tenant.tenant_id`
- `hz_contract.house_id` → `hz_house.house_id`

### 5.3 数据状态

所有业务表都有 `del_flag` 字段：
- `0` = 正常数据
- `2` = 已删除数据

导出时默认过滤了已删除数据 (`WHERE del_flag = '0'`)
