## 二、后端部署（自动）

### 触发方式
`git push origin master` 且 `ruoyi-**/**` 或 `pom.xml` 有改动时自动触发。

### 流程
```
git push → GitHub Actions 触发
         → SSH 登录后端服务器
         → 服务器从 Gitee 拉最新代码（git pull）
         → mvn clean package（本地编译，约3-5分钟）
         → 替换 JAR → supervisorctl restart
```

### 关键路径
| 路径 | 说明 |
|------|------|
| `/www/wwwroot/ghz-source/` | Gitee 代码目录 |
| `/www/wwwroot/ghz-backend/ruoyi-admin.jar` | 运行中的 JAR |
| `/www/wwwroot/ghz-backend/ruoyi-admin-backup.jar` | 上一个版本备份 |
| `/www/wwwroot/ghz-backend/logs/app.log` | 应用运行日志 |
| `/www/wwwroot/ghz-backend/logs/deploy.log` | 部署日志 |
| `/www/wwwroot/ghzupload/` | 文件上传存储目录 |

### 查看部署日志
```bash
# SSH 登录后端服务器
ssh -p 3322 deploy@36.133.39.49

# 查看部署进度
tail -f /www/wwwroot/ghz-backend/logs/deploy.log

# 查看应用运行日志
tail -f /www/wwwroot/ghz-backend/logs/app.log

# 查看服务状态
sudo supervisorctl status ghz-backend
```

### 手动重启
```bash
sudo supervisorctl restart ghz-backend
```

### 手动触发部署（不推送代码时）
```bash
ssh -p 3322 -i ~/.ssh/github_deploy deploy@36.133.39.49 \
  "bash /www/wwwroot/ghz-source/server-deploy.sh"
```

## 四、H5 用户端部署（手动）

> H5 使用 HBuilderX 开发，无法通过 GitHub Actions 自动构建，每次发版手动操作。

### 步骤

**第一步：确认 config 是生产环境**

`uniapp-h5/config/index.js` 中：
```js
const env = 'production'  // 发版时必须是 production
```

**第二步：HBuilderX 打包**
```
HBuilderX → 发行 → 网站 PC Web（H5）
```
构建输出目录：`uniapp-h5/unpackage/dist/build/web/`

**第三步：运行部署脚本**
```bash
# 在项目根目录执行
bash uniapp-h5/deploy-h5.sh
```

### 关键路径
| 路径 | 说明 |
|------|------|
| `/opt/1panel/www/sites/app.caigon.cn/index/` | H5 文件目录 |

### 注意事项
- `unpackage/` 目录不要提交到 git（构建产物）
- 本地开发调试时将 `env` 改回 `test`，发版前再改回 `production`

项目同时维护 Gitee 和 GitHub 两个远程仓库：

> 服务器从 Gitee 拉代码（国内速度快）。

## 十、常见问题

**Q: 后端部署后服务没有重启？**
```bash
# 查看 Supervisor 状态
sudo supervisorctl status ghz-backend
# 手动重启
sudo supervisorctl restart ghz-backend
```

**Q: 管理后台刷新页面 404？**
```
1Panel → 网站 → admin.caigon.cn → 伪静态
填入：location / { try_files $uri $uri/ /index.html; }
```

**Q: 图片无法访问？**
- 确认文件在 `/www/wwwroot/ghzupload/` 对应子目录下
- 确认 `deploy` 用户有读写权限：`chown -R deploy:deploy /www/wwwroot/ghzupload/`

**Q: 后端编译失败怎么回滚？**
```bash
# 服务器上手动恢复备份
ssh -p 3322 deploy@36.133.39.49
cp /www/wwwroot/ghz-backend/ruoyi-admin-backup.jar \
   /www/wwwroot/ghz-backend/ruoyi-admin.jar
sudo supervisorctl restart ghz-backend
```

**Q: H5 访问的 API 地址不对？**
检查 `uniapp-h5/config/index.js` 中 `env` 是否为 `production`，重新打包部署。
