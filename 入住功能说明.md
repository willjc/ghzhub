# 入住功能实现说明

## 一、功能概述

本次更新实现了完整的入住办理流程，包括：
1. 签约后自动生成入住单（带入住单号）
2. 用户端在线办理入住（填写实际入住日期、合住人、紧急联系人）
3. 管理端审核入住申请
4. 完整的状态流转：待办理 → 待审核 → 已入住/已拒绝

## 三、状态流转说明

```
签约成功 → 生成入住单（status='0' 待办理）
    ↓
用户办理入住 → 提交详细信息（status='1' 待审核）
    ↓
管理员审核 → 通过（status='2' 已入住）或拒绝（status='3' 已拒绝）
```

| 状态值 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 待办理 | 合同签约后自动生成，等待用户办理 |
| 1 | 待审核 | 用户提交入住信息，等待管理员审核 |
| 2 | 已入住 | 管理员审核通过 |
| 3 | 已拒绝 | 管理员拒绝入住 |

## 五、前端开发指南

### UniApp H5 页面开发建议

#### 1. 入住申请列表页 (`/pages/affairs/checkin`)

**功能需求：**
- 显示用户的所有入住单
- 按状态分组显示（待办理、待审核、已入住、已拒绝）
- 点击"办理入住"按钮跳转到办理页面（仅待办理状态可点击）

**代码示例：**
```javascript
// 获取入住单列表
async loadCheckInList() {
  const tenantId = uni.getStorageSync('tenantId')
  const res = await uni.request({
    url: `http://localhost:8090/h5/app/checkin/list/${tenantId}`,
    method: 'GET'
  })
  this.checkInList = res.data.data
}

// 点击办理入住
handleCheckIn(recordId) {
  uni.navigateTo({
    url: `/pages/affairs/checkin-process?type=talent&id=${recordId}`
  })
}
```

#### 2. 入住办理页面 (`/pages/affairs/checkin-process`)

**功能需求：**
- 显示入住单基本信息（入住单号、房源信息、合同租期等）
- 选择实际入住日期（日期选择器）
- 填写合住人信息（可添加多个）
  - 姓名
  - 身份证号
  - 联系方式
- 填写紧急联系人信息
  - 姓名
  - 关系（下拉选择）
  - 联系方式
- 提交按钮

**代码示例：**
```javascript
async loadCheckInDetail(recordId) {
  const res = await uni.request({
    url: `http://localhost:8090/h5/app/checkin/${recordId}`,
    method: 'GET'
  })
  this.checkIn = res.data.data
}

// 添加合住人
addRoommate() {
  this.roommates.push({
    name: '',
    idCard: '',
    phone: ''
  })
}

// 提交办理信息
async submitCheckIn() {
  const params = {
    recordId: this.recordId,
    actualCheckinDate: this.form.actualCheckinDate,
    roommateInfo: JSON.stringify(this.roommates),
    emergencyContactName: this.form.emergencyContactName,
    emergencyContactRelation: this.form.emergencyContactRelation,
    emergencyContactPhone: this.form.emergencyContactPhone
  }

  const res = await uni.request({
    url: 'http://localhost:8090/h5/app/checkin/submit',
    method: 'POST',
    data: params
  })

  if (res.data.code === 200) {
    uni.showToast({ title: '提交成功' })
    setTimeout(() => {
      uni.navigateBack()
    }, 1500)
  }
}
```

## 六、测试流程

### 1. 签约测试
1. 用户端选择房源 → 填写合同信息 → 确认签约
2. 检查数据库 `hz_checkin_record` 表是否生成入住单
3. 检查入住单号是否自动生成（格式：RZ开头）
4. 检查状态是否为 `0`（待办理）

### 2. 办理入住测试
1. 访问入住申请列表页，查看是否显示新入住单
2. 点击"办理入住"进入办理页面
3. 填写实际入住日期、合住人、紧急联系人信息
4. 提交后检查状态是否变为 `1`（待审核）

### 3. 审核测试
1. 管理端访问入住管理页面
2. 查看入住详情，确认显示用户填写的信息
3. 点击"审核通过"或"审核拒绝"
4. 检查状态是否更新，审核人和审核时间是否记录

## 七、注意事项

### 1. 数据库执行
- **必须先执行 SQL 脚本**才能运行后端代码
- 如果已有入住数据，SQL 脚本会为现有数据生成入住单号

### 2. 权限配置
管理端审核需要权限 `gangzhu:checkin:audit`，需在系统管理中配置。

### 3. 合住人信息格式
合住人信息使用 JSON 数组存储：
```json
[
  {"name":"张三","idCard":"410...","phone":"138..."},
  {"name":"李四","idCard":"410...","phone":"139..."}
]
```

### 4. 日期格式
- 实际入住日期：`YYYY-MM-DD` 格式（如：2025-01-21）
- 审核时间：`YYYY-MM-DD HH:mm:ss` 格式
